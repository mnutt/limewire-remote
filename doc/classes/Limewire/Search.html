Class Limewire::Search
In: lib/limewire.rb
Parent: Object

Searching in LimeWire is an incremental process: a search is created, then polled periodically to find the results that have been returned.

Methods

find   guid   new   query   query_string   raw_results   restart   results   start   stop  

Public Class methods

find(guid)

Fetches a search by its guid. This creates a new Search object from LimeWire‘s SearchWithResults object

  • guid: The identifier (as a String) of the search
new(search)
query(query)

Creates a new search based on a search term. Returns a Search, which provides a guid for lookup later.

  • query: The term to search for

Public Instance methods

guid()

Returns the search guid, which is a java Guid and needs to_s to be turned into a string guid.

query_string()

Retrieve the search term

raw_results()

The LimeWire Client‘s SearchWithResults java object

restart()

Clear the results and begin the search again.

results()

Results of the search, up until this point. The search is started on +self.query+, and runs until it is stopped. results can be called to get an updated list of search results. Results are returned as an array of hashes.

Example: 

  >> search = Search.query("grapefruit")
  >> results = search.results

  => { :filename => "grapefruit.txt",
       :magnet_url => "magnet:?urn:sha1:823A3F24DF34012BB35823A3F24DF34012BB35",
       :spam => false,
       :sha1 => "urn:sha1:823A3F24DF34012BB35823A3F24DF34012BB35",
       :properties => {} }
start()

Start looking for search results.

stop()

Stop searching for results.
